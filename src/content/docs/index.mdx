> The way **SAML** standardized "who is this user," and **MCP** standardized "what tools can an agent call" — **AGP** standardizes "what is this agent allowed to do, who said so, and what did it actually do" across every AI agent vendor.

## The problem AGP solves

Within 18–24 months a typical enterprise will run 50–200 AI agents across 10–30 vendors — each with its own audit format, its own policy surface (if any), and its own identity model. There is no unified way to see what every agent did, enforce one policy everywhere, or prove it to an auditor.

## Three flows, three conformance levels

<CardGrid>
  <Card title="L1 — Events" icon="document">
    Vendors emit **signed canonical events** for every agent action. Passive observability with verifiable provenance. Ships in **≤1 engineer-week**.
  </Card>
  <Card title="L2 — Governance" icon="approve-check">
    Vendors also **accept and apply customer policy**. One policy DSL, enforced inside every conformant vendor.
  </Card>
  <Card title="L3 — Real-time" icon="rocket">
    Vendors **call back to the plane synchronously** to gate high-stakes actions in &lt;300ms. Human-in-the-loop becomes a protocol primitive.
  </Card>
</CardGrid>

## Built in the open, verifiable end-to-end

- **Cryptographic by default.** Events and policies are Ed25519-signed and canonicalized with RFC 8785 ([ADR 0001](/spec/adr-0001/)). Auditors verify signatures without trusting any vendor.
- **Three reference implementations agree on every byte.** Python, TypeScript, and a Go conformance suite — `pip install openagp`, `npm install @openagp/sdk`, `agp-cts vectors`.
- **A public registry** of conformant actors and their public keys.
